              attached to the learned code signals.

              Use Analyze as the main path: paste code, analyze it, then inspect
              the latest learned signals.
            
                                          3. 
            
                                        3. 
              Open Tools only when you need formatting, syntax preview, or
              editor cleanup.
            
                                          4. 
            
                                        4. 
              Use Knowledge for both saved sources and profile management, then
              use Studio for turning that knowledge into concrete product
              direction.
            
                                          5. 
            
                                        5. 
              Use the Profile section inside Knowledge when you need deeper
              basics editing, source curation, or advanced artifacts like
              exports and memory.
